Componentes reutilizables, React Router y pantalla de detalle

Concepto 1

Componente reutilizable con props tipadas

Un componente en React es una función que recibe datos (props) y devuelve JSX. Al tiparlo con una interface de TypeScript, el compilador te avisa si olvidas pasar algún dato o si lo mandas del tipo equivocado. 

// Interfaz de props — define el contrato del componente
interface PokemonCardProps {
  name: string;
  url:  string;
}

// El componente recibe esas props y las desestructura
function PokemonCard({ name, url }: PokemonCardProps) {
  return <div>{name}</div>;
}
Cada tarjeta es independiente: recibe su name y hace su propio fetch. Así HomePage solo lista nombres y las tarjetas se llenan solas en paralelo.
Si pones toda la lógica directo en HomePage sin extraer el componente, violas RT08 (no poner todo en un solo archivo) y el código se vuelve imposible de reutilizar en otras páginas.
Concepto 2

useEffect para cargar datos al montar

useEffect ejecuta código con efectos secundarios (como llamadas a API) después de que el componente se renderiza. El array de dependencias controla cuándo vuelve a ejecutarse. 

useEffect(() => {
  getPokemonDetails(name)
    .then((data) => {
      setPokemon(data);
      setLoading(false);
    })
    .catch(() => setLoading(false));
}, [name]); // ← se re-ejecuta si "name" cambia
Si dejas el array vacío [], el efecto solo corre una vez al montar. Si la tarjeta recibe un name diferente más adelante (búsqueda, filtros), nunca actualizará los datos.
Concepto 3

React Router — Rutas y navegación

React Router v7 mapea URLs a componentes. Vive en App.tsx y envuelve toda la app. Con :id en la ruta defines un parámetro dinámico que cada página puede leer con useParams. 

// App.tsx
<BrowserRouter>
  <Routes>
    <Route path="/"            element={<HomePage />}   />
    <Route path="/pokemon/:id" element={<DetailPage />} />
  </Routes>
</BrowserRouter>
HerramientaPara qué sirve
BrowserRouterProveedor de router — va en la raíz
Routes / RouteDefine qué componente renderiza cada URL
<Link to="...">Navega sin recargar la página (como un <a> inteligente)
useParams()Lee el parámetro :id de la URL actual
Usa <Link> en lugar de <a href>. Un <a> normal recarga toda la página y pierde el estado de React.
Concepto 4

DetailPage — Leer parámetros de URL

useParams devuelve un objeto con los parámetros dinámicos de la ruta actual. Si la ruta es /pokemon/:id y la URL es /pokemon/25, entonces id vale "25". 

function DetailPage() {
  const { id } = useParams<{ id: string }>();

  useEffect(() => {
    if (!id) return;
    getPokemonDetails(id).then(...)
  }, [id]);
}
